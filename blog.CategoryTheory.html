Category Theory blog post

Category theory is a fascinating framework of mathematics that is used to study all kinds of compositional structures. My first contact with category theory was the famous phrase “a monad is just a monoid in the category of endofunctors lol”. This elusive framework looked like wizardry to me which was attractive and convinced me to start picking it up bit by bit. In this post, I aim to define important concepts in category theory bit by bit while giving plenty of examples for each new concept, thus keeping the material as concrete as abstract nonsense can be.

I will also be using agda to formally define (and prove) everything I say. There is already a very nice online formalisation of (univalent) category theory at https://1lab.dev/, but the material is scattered, too decentralised and lacks the overarching narrative that a long linear blog post makes possible.

1. Category

A category is a tuple containing the following things: 

record Category (ℓₒ ℓₕ : Level) : Set (lsuc (ℓₒ ⊔ ℓₕ)) where

A set of objects 

  field
    ob : Set ℓₒ

For each pair of objects, a set of morphisms . 

  field
    hom : (X Y : ob) -> Set ℓₕ

For each object, an “identity” arrow that will serve as the left and right unit of composition. 

  field
    id : (X : ob) -> hom X X

For each pair of morphisms and , a composition operator . that “glues” arrows together when their endpoints match. 

  field
    _∘_ : {X Y Z : ob} -> (g : hom Y Z) -> (f : hom X Y) -> hom X Z

Moreover, this composition operator must have a left and right identity and be associative. 

  field
    id-left : {X Y : ob} {f : hom X Y} -> id Y ∘ f  ≡ f
    id-right : {X Y : ob} {f : hom X Y} -> f ∘ id X ≡ f

    assoc : {W X Y Z : ob}
      -> (f :  hom W X)
      -> (g : hom X Y)
      -> (h : hom Y Z)
      -> (h ∘ g) ∘ f ≡ h ∘ (g ∘ f)


variable
  ℓₒ ℓₕ : Level
  C : Category ℓₒ ℓₕ

What this mean is that a category holds a possibly infinite set of points we call objects

1.1.1. Category of sets and functions between sets.

The first category we are going to see (and the most useful one) is the category of sets. The objects are sets themselves and the morphisms are function between sets. This category can be thought of as the category of typed complete (always terminate) purely functional programs and because this category is so familiar (abnd because it possesses so much structure), we will use it a lot for concrete examples of abstract principles. 

  CatSet : {ℓₒ : Level} -> Category (lsuc ℓₒ) ℓₒ

The objects are set, so the “object set” is the set of sets. 

  (CatSet {ℓₒ}).ob = Set ℓₒ

The set of arrows between two sets is the set of functions between those two sets. 

  (CatSet {ℓₒ}).hom X Y = X -> Y

Composition of function is normal function composition. 

  (CatSet {ℓₒ})._∘_ g f x = g (f x)

For any set X, there is always a function f(x) = x that does nothing. 

  (CatSet {ℓₒ}).id X x = x

This identity is trivially a left unit ofr composition and the composition operator is associative. 

  (CatSet {ℓₒ}).id-left = refl
  (CatSet {ℓₒ}).id-right = refl
  (CatSet {ℓₒ}).assoc f g h = refl

  module CatSet = Category (CatSet {ℓₒ = lzero})

an example of a morphism that exists in this category is the suc function that takes a natural number and returns its successor. 

  some-morphism : CatSet.hom Nat Nat
  some-morphism = suc

Because its source and target objects are the same, we can compose it with itself a bunch of times 

  some-morphism² : CatSet.hom Nat Nat
  some-morphism² = CatSet._∘_ some-morphism some-morphism

and it will give the correct result . 

  some-morphism²-is-correct : some-morphism² 2 ≡ 4
  some-morphism²-is-correct = refl

1.2. Order category of the natural numbers

This is a very simple category which serves as an example of how even objects that are not “process-like” can be viewed through the lens of category theory. Before we can properly prove all the properties of the order on , we need to prove some lemmas first. 

  -- we need to show a number is never bigger than its successor
  sa≤a-false : {a : Nat} -> suc a ≤ a -> ⊥
  sa≤a-false {a} p = absurd (thing2 thing1)
    where
      thing1 : suc a ≡ a
      thing1 = ≤-sandwitch p (osuc orefl)

      thing2 : suc a ≡ a -> ⊥
      thing2 ()

  ≤-prop : {a b : Nat} -> (p q : a ≤ b) -> p ≡ q
  ≤-prop {a} {b} _≤_.orefl _≤_.orefl = refl
  ≤-prop {a} {b} _≤_.orefl (_≤_.osuc q) = absurd (sa≤a-false q)
  ≤-prop {a} {b} (osuc p) orefl = absurd (sa≤a-false p)
  ≤-prop {a} {b} (osuc p) (osuc q) = cong osuc (≤-prop p q)

  CatNat : Category lzero lzero
  CatNat .ob = Nat
  CatNat .hom x y = x ≤ y
  CatNat ._∘_ Y≤Z X≤Y = ≤-trans X≤Y Y≤Z
  CatNat .id X = _≤_.orefl
  -- All the composition identities, because there is only one way for a ≤ b
  CatNat .id-left = ≤-prop _ _
  CatNat .id-right = ≤-prop _ _
  CatNat .assoc p q r = ≤-prop _ _

The terminal object is the (unique up to isomorphism) object such that for any other object there is only a single arrow from tat object to our terminal object. Dually, the initial object is one for which there is a single outgoing arrow to any

3. Product and coproduct objects

In a category, the product between two objects can be thought of (for categories that look like preorders) “the greatest lower bound of and ”. Dually, the coproduct can be thought of as “the least upper bound” of and .

For a category with more structure, like the category of sets, the categorical product and coproduct give, respectively, the cartesian product and the disjoint union . 

record is-product {ℓₒ ℓₕ} {C : Category ℓₒ ℓₕ} {A B P : C .Category.ob} (π₁ : C .Category.hom P A) (π₂ : C .Category.hom P B) : Set (ℓₒ ⊔ ℓₕ) where
  open Category C
  field
    ⟨_,_⟩ : {Q : ob}
      -> (a : hom Q A)
      -> (b : hom Q B)
      -> hom Q P

    π₁∘⟨⟩ : {Q : ob}
      -> {a : hom Q A}
      -> {b : hom Q B}
      -> (π₁ ∘ ⟨ a , b ⟩) ≡ a

    π₂∘⟨⟩ : {Q : ob}
      -> {a : hom Q A}
      -> {b : hom Q B}
      -> (π₂ ∘ ⟨ a , b ⟩) ≡ b

    unique : {Q : ob}
      -> {other : hom Q P}
      -> {a : hom Q A}
      -> {b : hom Q B}
      -> π₁ ∘ other ≡ a
      -> π₂ ∘ other ≡ b
      -> other ≡ ⟨ a , b ⟩

Diagrammatically, the product between two objects is the triple such that for any other triple , there exists a unique morphism such that and . In other words, there is exactly one morphism that when joining to , make the diagram commute.

  min : Nat -> Nat -> Nat
  min zero b = zero
  min (suc a) zero = zero
  min (suc a) (suc b) = suc (min a b)

  min-comm : (a b : Nat) -> min a b ≡ min b a
  min-comm zero zero = refl
  min-comm zero (suc b) = refl
  min-comm (suc a) zero = refl
  min-comm (suc a) (suc b) = cong suc (min-comm a b)

  greatest-lower-bound : (a b c : Nat) -> c ≤ a -> c ≤ b -> c ≤ min a b
  greatest-lower-bound zero zero c orefl q = q
  greatest-lower-bound zero (suc b) c p q = p
  greatest-lower-bound (suc a) zero c p q = q
  greatest-lower-bound (suc a) (suc b) zero p q = z≤n
  greatest-lower-bound (suc a) (suc b) (suc c) p q = oshift (greatest-lower-bound a b c (ounshift p) (ounshift q))

  lower-bound : (a b : Nat) -> (min a b ≤ a) × (min a b ≤ b)
  lower-bound zero b = orefl , z≤n
  lower-bound (suc a) zero = z≤n , orefl
  lower-bound (suc a) (suc b) = oshift left , oshift right
    where
      left : min a b ≤ a
      left = lower-bound a b .fst
      right : min a b ≤ b
      right = lower-bound a b .snd

  min-is-product : {a b : Nat} -> is-product {C = CatExamples.CatNat} {P = min a b} (lower-bound a b .fst) (lower-bound a b .snd)

5. Pullback and Pushout

record is-pullback {ℓₒ ℓₕ} {C : Category ℓₒ ℓₕ} {X×Y X Y Z : C .Category.ob}
  (π₁ : C .Category.hom X×Y X)
  (π₂ : C .Category.hom X×Y Y)
  (f₁ : C .Category.hom X Z)
  (f₂ : C .Category.hom Y Z) : Set (ℓₒ ⊔ ℓₕ) where
  open Category C
  field

    -- The diagram commutes
    square : f₁ ∘ π₁ ≡ f₂ ∘ π₂

    -- Two arrows into the (commutative) diagram can be turned into a single
    -- (universal) arrow.
    universal : {P : ob} (π₁' : hom P X) (π₂' : hom P Y)
      -> f₁ ∘ π₁' ≡ f₂ ∘ π₂'
      -> hom P X×Y

    -- Which can be reverted to any of the two functions
    π₁∘universal : {P : ob} {π₁' : hom P X} {π₂' : hom P Y}
      -> (p : f₁ ∘ π₁' ≡ f₂ ∘ π₂')
      -> π₁ ∘ universal π₁' π₂' p ≡ π₁'

    π₂∘universal : {P : ob} {π₁' : hom P X} {π₂' : hom P Y}
      -> (p : f₁ ∘ π₁' ≡ f₂ ∘ π₂')
      -> π₂ ∘ universal π₁' π₂' p ≡ π₂'

    unique : {P : ob}
      -> (u : hom P X×Y)
      -> (π₁' : hom P X)
      -> (π₂' : hom P Y)
      -> (eqc : f₁ ∘ π₁' ≡ f₂ ∘ π₂')
      -> (eq1 : π₁' ≡ π₁ ∘ u)
      -> (eq2 : π₂' ≡ π₂ ∘ u)
      -> u ≡ universal π₁' π₂' eqc


module PullbackExamples where

  open is-pullback

  record Solution {ℓ} {X Y Z : Set ℓ} (f : X -> Z) (g : Y -> Z) : Set ℓ where
    constructor sol
    field
      π₁ : X
      π₂ : Y
      eq : f π₁ ≡ g π₂

  open Solution

  solution-is-pullback : {ℓ : Level} {X Y Z : Set ℓ} {f : X -> Z} {g : Y -> Z} -> is-pullback {C = CatExamples.CatSet} {X×Y = Solution f g} π₁ π₂ f g
  solution-is-pullback .square = extensionality λ { (sol x y eq) → eq}
  solution-is-pullback .universal π₁' π₂' eq p = sol (π₁' p) (π₂' p) (eq <≡> pure p)
  solution-is-pullback .π₁∘universal p = extensionality (λ some → refl)
  solution-is-pullback .π₂∘universal p = extensionality (λ some → refl)
  solution-is-pullback {f = f} {g} .unique u π₁' π₂' eqc eq1 eq2 = extensionality pointwise
    where
      lemma : {X Y Z : Set ℓ} (a b : X) (c d : Y) {f : X -> Z} {g : Y -> Z}
        -> (a=b : a ≡ b)
        -> (c=d : c ≡ d)
        -> (p : f a ≡ g c)
        -> (q : f b ≡ g d)
        -> sol a c p ≡ sol b d q
      lemma a b c d refl refl p q = pure≡ (sol a c) <≡> K p q
          where K : {A : Set ℓ} {x y : A} -> (p q : x ≡ y) -> p ≡ q
                K refl refl = refl

      pointwise some = u some ≡⟨⟩
                       sol (u some .π₁) (u some .π₂) (u some .eq) ≡⟨ lemma (u some .π₁) (π₁' some) (u some .π₂) (π₂' some) ((sym eq1) <≡> refl ) (sym eq2 <≡> refl) (u some .eq) (eqc <≡> refl) ⟩

6. Functors

We have seen how we can view many (two, to be exact) things as categories and describe structure in a very general way. Let’s give categories a taste of their own medecine by making them into objects of their own category!

Before we do that, we need a way to talk about arrows between categories. The usual way of studying some algebraic structure such as a group, a monoid or something else is through the category of s and homomorphisms. Category homomorphisms are called functors. 

record Functor {ℓₒ ℓₕ} (C D : Category ℓₒ ℓₕ) : Set (ℓₒ ⊔ ℓₕ) where
  constructor functor-constructor
  private
    module C = Category C
    module D = Category D

The pieces of data that make up a functor are the following.

First, we have a function that associates to every object in the source category a object in the target category. 

  field
    ob-map : C.ob -> D.ob

Then, we have a function that associates to each morphism in the source category a morphism in the target category. This mapping must respect the objects selected by ob-map. 

  field
    hom-map : {A B : C.ob}
      -> C.hom A B
      -> D.hom (ob-map A) (ob-map B)

Finally, the morphism mapping must preserve the identity morphisms and composition. 

    preserves-id : {A : C.ob}
      -> hom-map (C.id A) ≡ D.id (ob-map A)

    preserves-∘ : {A B C : C.ob} {f : C.hom A B} {g : C.hom B C}
      -> hom-map (g C.∘ f) ≡ (hom-map g D.∘ hom-map f)

module Functors where
  open Category
  open Functor


  Functor-id : (C : Category ℓₒ ℓₕ) -> Functor C C
  Functor-id C .ob-map x = x
  Functor-id C .hom-map f = f
  Functor-id C .preserves-id = refl
  Functor-id C .preserves-∘ = refl

  module FunctorComp {C D E : Category ℓₒ ℓₕ} (G : Functor D E) (F : Functor C D) where
    private
      module C = Category C
      module D = Category D
      module E = Category E

    comp : Functor C E
    comp .ob-map X = G .ob-map (F .ob-map X)
    comp .hom-map {A} {B} m = G .hom-map (F .hom-map m)
    comp .preserves-id {A} =
      G .hom-map (F .hom-map (C .id A)) ≡⟨ cong (G .hom-map) (F .preserves-id) ⟩
      G .hom-map (D .id (F .ob-map A)) ≡⟨ G .preserves-id ⟩
      id E (G .ob-map (F .ob-map A)) ∎
    comp .preserves-∘ {f = f} {g} =
      G .hom-map (F .hom-map (g C.∘ f)) ≡⟨ cong (G .hom-map) (F .preserves-∘) ⟩
      G .hom-map ((F .hom-map g) D.∘ (F .hom-map f)) ≡⟨ G .preserves-∘ ⟩
      (G .hom-map (F .hom-map g)) E.∘ (G .hom-map (F .hom-map f)) ∎

  open FunctorComp

  module FunctorAssoc {A B C D : Category ℓₒ ℓₕ} (F : Functor A B) (G : Functor B C) (H : Functor C D) where

    K : {A : Set ℓ} {x y : A} (p q : x ≡ y) -> p ≡ q
    K refl refl = refl

    obeq : (comp (comp H G) F).ob-map ≡ (comp H (comp G F)).ob-map
    obeq = refl

    homeq : (λ {A} {B} -> (comp (comp H G) F).hom-map {A = A} {B}) ≡ (λ {A} {B} -> (comp H (comp G F)).hom-map {A} {B})
    homeq = refl

    id-eq : (λ A -> (comp (comp H G) F).preserves-id {A}) ≡ (λ A -> (comp H (comp G F)).preserves-id {A})
    id-eq = extensionality λ some → K _ _

    id-comp : (λ A B C f g -> (comp (comp H G) F).preserves-∘ {A} {B} {C} {f} {g}) ≡ (λ A B C f g -> (comp H (comp G F)).preserves-∘)
    id-comp = extensionality λ some → extensionality (λ some₁ → extensionality λ some₂ → extensionality λ some₃ → extensionality λ some₄ → K _ _)

open Functors
